## Scope

Information architecture and UX of doulab.net post-ES-launch (commit `eb1c8c8`). Covered:

- Navbar + footer IA in both locales (EN at `/`, ES at `/es/*`).
- Page-to-page flow on top-level routes and on the `/services/*` and `/case-studies/*` deep trees.
- Sidebar logic for `/docs/*` and the `intro` vs `research-resources` entry-point split.
- Locale-switching round-trip behavior: dropdown behavior, link resolution under `/es/*`, parity of route trees, canonical / hreflang correctness.
- Breadcrumb status (visible component vs JSON-LD only).
- Orphan pages / dead-end routes / redirects (`_redirects`).
- Sitemap clarity (sitemap-per-locale, hreflang, navbar/footer surface inclusion).
- Parity drift between EN and ES surfaces: routes present in EN but missing in ES (or vice versa), copy chrome that EN-falls-through in ES, glossary drift carried over from E-I2-R1.

Visual / brand polish, copy tone, conversion microcopy, SEO mechanics, a11y SVG roles, and security are out of scope — handed off to the matching role files.

## Quick wins (top 5, ≤1 day each)

1. **IAUX-101 (P0)** — Fix all 24 ES `<link rel="canonical">` declarations to point at `https://www.doulab.net/es/<path>` instead of `https://doulab.net/<path>`. Strip the host inconsistency at the same time. Single bulk-edit pass. Unblocks ES indexing.
2. **IAUX-103** — Add Vigía Futura to the navbar (`docusaurus.config.ts:130-136`) and add the matching `item.label.Vigía Futura` key to `i18n/es/docusaurus-theme-classic/navbar.json`. Two-file change, no rebuild risk.
3. **IAUX-105** — Replace `terms-and-conditions.tsx` React stubs (EN + ES) with `_redirects` entries. Delete both files.
4. **IAUX-112** — Add a "looking for the other language?" banner to both 404 pages. Two small edits.
5. **IAUX-106** — Move `releases` out of `whitepaperSidebar` in `sidebars.ts`. One-line edit + update the footer "More > Releases" link if path changes.

## Strategic bets (top 3)

1. **IAUX-102 (visible breadcrumbs sitewide)** — Build a real `<Breadcrumb>` component driven by the same data that already feeds the JSON-LD schema. Render on every second-level+ page in both locales. Resolves wayfinding for the new 3-deep `/services/clarityscan/{audit,diagnostic}` tree and for every deep `/case-studies/*` and `/services/*` route. Closes a 2-audit-old finding and prepares the IA for further depth growth.
2. **IAUX-107 + IAUX-108 (right-nav role split + footer restructure)** — Decide the role split between `/contact` and `/work-with-us`, then restructure the footer to four conventional columns (Solutions / Learn / Company / Legal & Feeds). Both touch the most-visited chrome and are the single highest-leverage IA cleanup left after the E-H1 consolidation. Mirror in ES chrome JSON.
3. **IAUX-115 (page-content data extraction)** — Move large pages (home, about, services hub, IMM-P®, IMM-DT, ClarityScan, Vigía Futura, all 4 case studies) from EN/ES TSX-duplicated content into locale-keyed JSON/MDX data + thin React shells. Eliminates the editorial-drift risk that the current 600-700-LOC-per-page-per-locale pattern bakes in. Largest single-effort win for long-term bilingual maintenance.
